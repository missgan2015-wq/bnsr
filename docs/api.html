easyLua API 参考

easyLua 是基于 LuaJIT 5.4 + Android root 的自动化引擎。所有 API 通过 LuaJIT FFI 直接调 native 实现。

命名规则：PascalCase，所有 API 都是顶层全局函数，脚本启动即可用，无需 require 或 ffi.cdef。

下方按功能分组（Screen / Images / Motion 等仅作为分类标签，不是命名空间前缀），调用时直接写函数名即可。例如 FindColor(...)、Click(x, y)、Toast("hi")。

快速开始

-- 直接读屏（首次会自动等首帧到达，无需手动等） local x, y = FindColor(0, 0, 0, 0, "FF3344-101010", 0.95, 0) if x >= 0 then Highlight(x - 30, y - 30, 60, 60, "#FF0000", 2000, "找到了") Click(x, y) Toast("已点击 (" .. x .. ", " .. y .. ")") end

Screen 屏幕信息 + 像素读取

ScreenWidth()
当前帧的实际像素宽度（旋转后逻辑宽度）。
返回: int
ScreenHeight()
当前帧的实际像素高度。
返回: int
ScreenSeq()
帧序号。每收到一帧 +1，可用于检测"画面是否变化"。
返回: int
Pixel(x, y)
读取像素的 RGBA 4 通道值（0-255）。越界返回 0,0,0,0。
返回: r, g, b, a
PixelHex(x, y)
读取像素并格式化为 "RRGGBB" 字符串。
返回: string
无需手动等首帧：所有读屏 API（Width / Height / Pixel / Find* / Cmp* / GetColorCountInRegion） 在视频流首帧未到达时会自动 block 等待最多 3 秒。脚本可以"上来就用"。

CaptureScreen 帧快照

CaptureScreen 把当前帧的指定区域复制到一份独立 RGBA buffer， 返回一个**冻结**的快照对象。即使后续视频流推了新帧，也不会影响这份数据。 适用于：截图存盘、跨多帧对比、OCR / 模板匹配等耗时处理。

CaptureScreen(x1, y1, x2, y2)
抓取区域 (x1, y1)-(x2, y2) 的快照。x2 / y2 = 0 或省略表示用屏幕边界， 完全省略所有参数即全屏抓取。
返回: table - 快照对象，挂有 下方方法表。失败时返回 nil （视频流首帧未到 / 区域非法 / 内存不足）。
底层 RGBA buffer 由 LuaJIT GC 自动回收；也可显式调 :Free()， 或用 Lua 5.4 <close> 注解让作用域退出立即释放。

快照对象方法

方法 说明
snap:Width() 快照宽度。返回 int。
snap:Height() 快照高度。返回 int。
snap:Pixel(x, y) 取像素 RGBA 4 通道值（0-255）。越界返回 0, 0, 0, 0。返回 r, g, b, a。
snap:PixelHex(x, y) 取像素 "RRGGBB" 字符串。返回 string。
snap:Save(path, type, quality) 把快照保存到文件。
  • path：输出路径
  • type："png" / "jpg"（亦作 "jpeg"）/ "bmp"， 省略时按 path 后缀自动推断（识别 .png / .jpg / .jpeg / .bmp）
  • quality：仅对 JPG 生效，1..100；省略 / <= 0 取默认 90
返回 bool，未识别格式或写文件失败时返回 false。
snap:Free() 立即释放底层 buffer，不必等 GC。释放后所有访问返回零值，重复调用安全。
自动释放：以下三种方式都不需要手动调 :Free()。
  • GC 兜底：脚本不持有引用后，下次 GC 触发时底层 buffer 自动归还，不会泄漏。
  • <close> 注解（推荐）：作用域退出（do block 结束、函数 return、error 抛出）立即释放，不等 GC。 
    local snap <close> = CaptureScreen()
snap:Save("/sdcard/x.png")
-- 函数返回时 snap 立即释放
  • 显式 :Free()：精确控制时机，最直观。
-- 全屏抓帧 local snap = CaptureScreen() print(snap:Width(), snap:Height()) -- 抓某个区域并跨帧使用 local sub = CaptureScreen(100, 200, 400, 600) Sleep(500) -- 等动画结束，屏幕已经变了 local r, g, b = sub:Pixel(0, 0) -- 仍是抓取时的颜色 sub:Free() -- 显式释放（可省） -- 一行截屏存盘（type 按后缀推断） CaptureScreen():Save("/sdcard/shot.png") CaptureScreen():Save("/sdcard/shot.jpg", nil, 85)

Images 找色 / 比色

颜色字符串格式

格式 含义 例
RRGGBB 精确颜色 "FF3344"
RRGGBB-RRGGBB 带容差，后半段为 R/G/B 各通道允许偏移 "FF3344-101010"
C1|C2|C3 多候选 OR（命中任一即可） "FF0000|00FF00|0000FF"
x,y,c,x,y,c,... 多点 AND（DetectsMultiColors 用） "100,200,FF0000-101010"
base,dx,dy,c,... 基色+相对点（FindMultiColors 用） "FF3344,7,0,EE2233-101010"

sim 参数（0~1.0）会再附加 (1-sim)*255 容差到所有通道。

方向参数 dir（FindColor / FindMultiColors）

dir 扫描顺序
0 左→右，上→下（默认）
1 右→左，上→下
2 左→右，下→上
3 右→左，下→上

API

CmpColor(x, y, color, sim)
测试单点是否匹配颜色。color 可以是单值 / 带容差 / 多候选。
返回: bool
FindColor(x1, y1, x2, y2, color, sim, dir)
在区域内找颜色，返回首个命中坐标。x2/y2 = 0 表示用屏幕边界。
返回: x, y - 找不到返回 -1, -1
GetColorCountInRegion(x1, y1, x2, y2, color, sim)
统计区域内符合颜色的像素数量。
返回: int
DetectsMultiColors(colors, sim)
多点比色：所有点位都命中才返回 true。
colors 格式: "x1,y1,RRGGBB-tol,x2,y2,..."
返回: bool
FindMultiColors(x1, y1, x2, y2, colors, sim, dir)
基色全屏扫描 + 相对偏移点验证。基色命中后再校验所有相对点全部命中才算找到。
colors 格式: "base[-tol],dx1,dy1,c1[-tol],dx2,dy2,c2[-tol],..."
返回: x, y - 找不到返回 -1, -1
FindMultiColorsAll(x1, y1, x2, y2, colors, sim, dir)
找区域内所有匹配位置。每命中一个会屏蔽 bbox 矩形避免重复。最多 256 个。
返回: { {x=,y=}, ... }
截屏存盘

easyLua 不直接在找色这块提供存盘 API。请用 CaptureScreen() 抓帧得到快照对象， 再调 :Save(path, type, quality) 写入文件。type 省略时按 path 后缀推断，支持 png / jpg / bmp。

-- 全屏存 PNG（按后缀推断 type） CaptureScreen():Save("/sdcard/shot.png") -- 全屏存 JPG，质量 85 CaptureScreen():Save("/sdcard/shot.jpg", nil, 85) -- 抓某区域，显式指定 type CaptureScreen(100, 200, 400, 600):Save("/sdcard/sub.bmp", "bmp")
使用提示：
  • 选择罕见 / 高对比度的基色（如发光按钮、纯色 logo），避免黑 / 白 / 灰
  • 多点（≥3）相对点可以把误判降到几乎 0
  • 定位过坐标后，用 DetectsMultiColors 在循环里持续验证；不需要每次都重新 Find

找图 LoadTemplate / FindPic / FindPicAll

基于两段式 SAD（绝对差和）的模板匹配：先用最多 12 个高对比特征点稀疏预筛淘汰大多数候选位置，再对全部 valid 像素累加 SAD 并按相似度阈值早退。 相比找色，找图能直接以图片为目标，适合按钮图标、UI 元素等"形状已知"的场景。

透明像素自动识别：
  • Alpha 通道：PNG 中 alpha < 128 的像素视为透明，不参与匹配。
  • 四角同色 color key：PNG / BMP 模板四角四像素 RGB 严格相等时，自动把该颜色视作背景透明键（JPG 不启用，避免压缩噪点误识别）。
  • 透明像素既不进 SAD 累加，也不算入相似度分母。零配置即可处理"非矩形"按钮图标。

API

LoadTemplate(path)
从文件加载模板并完成预处理（解码 + 透明判定 + 特征点选取）。一次加载，多次复用。支持 .png / .bmp / .jpg，加密资源（.enc）会自动透明解密。
参数 说明
path 模板图片路径。绝对路径或相对脚本目录均可。
返回: tpl - 模板对象。失败返回 nil（文件不存在 / 解码失败 / 全透明 / 尺寸越界 / 内存不足）。
底层 cdata 由 LuaJIT GC 自动回收；也可显式调 tpl:Free()，或用 Lua 5.4 <close> 注解让作用域退出立即释放。

模板对象方法

方法 说明
tpl:Width() 模板宽度。返回 int。
tpl:Height() 模板高度。返回 int。
tpl:ValidPx() 有效像素数（剔除透明后参与匹配的像素数）。返回 int，调试用。
tpl:Free() 立即释放模板。释放后不应再传给 FindPic / FindPicAll，重复调用安全。
FindPic(x1, y1, x2, y2, tpl, sim, dir)
在区域内查找模板的左上角坐标，返回首个命中位置。x2 / y2 = 0 表示用屏幕边界。扫描方向语义与 FindColor 一致。
参数 默认值 说明
x1, y1, x2, y2 — 扫描区域，x2 / y2 = 0 表示扩展到屏幕边界。
tpl — 三种类型可选：
  • 字符串文件名（推荐）：如 "1.png"，自动按 Templates.dir 解析并缓存
  • LoadTemplate 返回的模板对象
  • Templates.get(name) 返回的模板对象
sim 0.9 相似度阈值，[0, 1]。1.0 要求像素完全一致；0.9 适合一般 UI 匹配。
dir 0 扫描方向 0..3，参考上方方向参数表。非法值视为未命中。
返回: x, y - 模板左上角坐标。找不到 / 模板比扫描区还大 / 帧未就绪 / 模板文件不存在时返回 -1, -1。
FindPicAll(x1, y1, x2, y2, tpl, sim)
找区域内所有匹配位置。每命中一个会屏蔽该模板尺寸的矩形避免重复。最多 256 个屏蔽矩形（命中数本身不限于 256）。
参数 默认值 说明
x1, y1, x2, y2 — 扫描区域，x2 / y2 = 0 表示扩展到屏幕边界。
tpl — 支持字符串文件名（按 Templates.dir 解析）或 LoadTemplate / Templates.get 返回的模板对象。
sim 0.9 相似度阈值，[0, 1]。
返回: { {x=, y=}, ... } - 命中点数组（左上角坐标）。无命中返回空表 {}。

Templates（模板缓存）

FindPic / FindPicAll 接受字符串文件名时，背后走 Templates 缓存层：首次自动 LoadTemplate，之后纯查表。脚本里通常完全不用感知，只有需要切换素材根目录或主动预加载时才用得上下面的 API。

字段 / 方法 说明
Templates.dir 当前模板搜索根目录。默认值由 _easyLua_bootstrap.entry 推导（脚本所在目录），找不到则回退到 /data/local/tmp/easyLua/scripts 或 /data/local/tmp/easyLua。
Templates.set_dir(new_dir) 切换搜索根目录，并清空已缓存的模板（旧模板对象保持有效，直到 GC）。返回 true 或 false, err。
Templates.get(name) 按文件名取（或加载并缓存）模板对象。解析顺序：原样路径 → Templates.dir/name → 子目录递归（最多 4 层）。失败返回 nil, err。
Templates.preload() 遍历 Templates.dir 加载所有 .png / .bmp / .jpg / .jpeg（递归最多 6 层）。返回 ok_n, fail_n。脚本启动时调用一次可消除首次 FindPic 的 IO 抖动。
解析路径：FindPic(0, 0, 0, 0, "1.png") 找文件的顺序——
  1. 原样路径（绝对路径或相对 cwd）
  2. Templates.dir/1.png
  3. Templates.dir 子目录递归（最多 4 层）
命中即缓存，后续同名文件直接零开销查表。Templates.set_dir(new) 会清空缓存，但旧模板对象本身仍可继续使用。
生命周期与 <close>：
  • 持久复用：把 tpl 放进 upvalue 或全局，整个脚本周期复用，避免重复解码。
  • 临时模板：用 Lua 5.4 <close> 注解让作用域退出立即释放，不必等 GC。
do
  local tmp <close> = LoadTemplate("/sdcard/cache/captured.png")
  Log("valid=" .. tmp:ValidPx())
end -- tmp 立即释放
-- 推荐用法：直接传文件名字符串，背后走 Templates 缓存 -- assets/scripts/<name>/btn_ok.png 部署后会出现在 Templates.dir 同级 local x, y = FindPic(0, 0, 0, 0, "btn_ok.png") if x ~= -1 then Click(x + 40, y + 40) end -- 启动时主动加载所有图（消除首次 FindPic 的 IO 抖动） Templates.preload() -- 切到自定义素材库 Templates.set_dir("/sdcard/MyAssets") local btn = Templates.get("1.png") print(btn:Width(), btn:Height(), btn:ValidPx()) -- 老用法仍兼容：手动 LoadTemplate local btn2 = LoadTemplate("ui/btn_ok.png") if not btn2 then Toast("找不到模板"); return end local x2, y2 = FindPic(0, 0, 0, 0, btn2, 0.95, 1) -- 找全部，批量点击 for _, p in ipairs(FindPicAll(0, 0, 0, 0, "btn_ok.png")) do Click(p.x + 40, p.y + 40) Sleep(80) end
性能与精度建议：
  • 模板尺寸越小越快。1080p 全屏单线程下：50×50 ≤ 15ms，100×100 ≤ 30ms，200×200 ≤ 60ms。
  • sim 越高越快（早退更激进），但容错越低。一般 UI 取 0.9，要求精确取 0.95+。
  • JPG 模板会被强制禁用 color key（避免压缩噪点误识别），如果需要透明背景请用 PNG。
  • 模板比扫描区还大时直接返回失败，不会越界扫描。

Motion 触摸事件注入（uinput）

需要 root：触摸通过 /dev/uinput 注入虚拟触摸设备。首次调用 Click / Swipe / TouchDown 等时会自动初始化，无需手动调。
Click(x, y, duration) 阻塞 root
单点点击。duration 默认 50ms，按下到抬起的间隔。
返回: bool
LongClick(x, y, duration) 阻塞 root
长按。等价 Click(x, y, duration or 800)。
Swipe(x1, y1, x2, y2, duration) 阻塞 root
直线滑动。duration 默认 300ms。
SwipeBezier(x1, y1, x2, y2, duration) 阻塞 root
贝塞尔曲线滑动，更接近真人。

多指 / 自定义手势

TouchDown(x, y, finger) root
指定手指按下。finger 是 slot（0-9，默认 0）。配合 TouchMove / TouchUp 自定义手势。
TouchMove(x, y, finger) root
指定手指移动到新位置。
TouchUp(finger) root
抬起指定手指。
-- 自定义手势：双指捏合（缩小） TouchDown(300, 800, 0) -- 手指 0 起点 TouchDown(700, 800, 1) -- 手指 1 起点 for i = 1, 10 do TouchMove(300 + i * 10, 800, 0) TouchMove(700 - i * 10, 800, 1) Sleep(20) end TouchUp(0) TouchUp(1)

Ui 屏幕浮窗

架构说明：
  • Highlight：root 进程内 Canvas 自绘，独立 WindowManager 浮窗，不依赖任何 APK
  • Toast：通过 am broadcast 触发 SampleUI APK 的 ToastReceiver 弹原生 Toast。需要安装 SampleUI APK，但 APK 不需要在前台运行
Toast(msg, x, y, dur) 需 SampleUI
显示系统原生 Toast。x, y 当前实现忽略（系统 Toast 不支持精确坐标），dur < 3500 用 SHORT，否则 LONG。
Highlight(x, y, w, h, color, dur, label) 异步
显示空心矩形高亮（SikuliX 风格）。多个 highlight 可同时存在，到时间自动消失。
color: 字符串 "#RRGGBB" / "#AARRGGBB" 或数字 ARGB；省略 = 红色
dur: 显示时长 ms（默认 2000，<= 0 表示常驻直到 HighlightOff）
label: 矩形上方的文字标签（可省）
HighlightOff()
立即清掉所有 highlight。

Dialog 启动前配置弹窗

工作机制：Dialog 不是脚本运行时调用的 API，而是脚本启动前由 SampleUI APK 弹出的配置对话框， 让用户先填好账号、模式、阈值等参数，再 fork 脚本进程。
  • 在脚本同级放一个 ui.lua（声明式 DSL），SampleUI 会在用户点 ▶ 时解析它并弹出 AlertDialog
  • 用户点"确定"后，填写的值会回写到 ui.lua 自身的 default 参数，下次启动直接看到上次的选择
  • 脚本里直接解析 ui.lua 就能拿到当前生效的配置（不再走中间的 JSON 文件）
  • 用户点"取消"则脚本不启动，悬浮球回到 ▶ 停止态
  • 没有 ui.lua 文件时直接跳过，不弹任何窗口

每种控件最多接收 4 个参数。带 varName 的控件值会被回写到 ui.lua 自身的 default 参数；没有 varName 的（如 TextView）只是装饰。

addSeparator()
水平分割线。
addCheckBox(varName, label, default)
复选框。结果类型：boolean。
addEditText(varName, label, default)
文本输入框。结果类型：string。
addEditNumber(varName, label, default)
数字输入框（接受小数 / 负数）。结果类型：number。
addSpinner(varName, label, options, default)
下拉选择。options 是字符串列表 {"快速","普通","慢速"}，default 是其中之一。结果类型：string（被选中的文本）。

单选组。参数同 addSpinner。结果类型：string。
addSeekBar(varName, label, min, max, default)
滑块。addSlider 是兼容别名。结果类型：number。
addProgressBar(varName, label, min, max, default)

addDatePicker(varName, label, default)
日期选择器。default 格式 "yyyy-MM-dd"，留空 = 今天。结果类型：string。
addTimePicker(varName, label, default)
时间选择器。default 格式 "HH:mm"，留空 = 当前时间。结果类型：string。
addColorButton(varName, label, default)
颜色按钮，点击弹色板。default 格式 "#RRGGBB"。结果类型：string。
addImage(assetPath, widthDp, heightDp)
显示一张图片（来自部署目录或 assets）。装饰用，不收集结果。

lfs LuaFileSystem（内置 1.9.0，免 require）

lfs 已经静态编进 libeasylua.so，启动时自动注册为全局表， 脚本里直接 lfs.xxx 就能用。require("lfs") 也兼容， 返回的是同一张表。完整 API 与官方 LuaFileSystem 1.9.0 对齐。

什么时候用 lfs：标准 io 只能读写已知路径的文件；目录遍历、 创建/删除目录、读修改时间、判断"这是文件还是目录"都得用 lfs。
lfs.currentdir()
取当前工作目录字符串。失败返回 nil, err。
lfs.chdir(path)
切换当前工作目录。返回 true 或 nil, err。
lfs.mkdir(path)
创建一级目录（不递归）。返回 true 或 nil, err。
lfs.rmdir(path)
删除空目录。非空目录会失败，请先清理子项。
lfs.dir(path) 迭代器
返回一个迭代器，每次取一个目录条目（包含 "." 与 ".."）。 出错抛 error，建议外面套 pcall。
lfs.attributes(path [, key])
取文件属性。不传 key 返回属性表；传字段名（如 "size" / "mode" / "modification"）返回单值。
属性表字段：
字段 含义
mode 类型："file" / "directory" / "link" / "socket" / "named pipe" / "char device" / "block device" / "other"
size 字节大小
modification 修改时间（Unix 秒）
access 访问时间
change 状态变更时间
permissions 权限字符串（如 "rwxr-xr-x"）
dev / ino / nlink / uid / gid / rdev / blocks / blksize POSIX 字段
lfs.symlinkattributes(path [, key])
类似 attributes，但不跟随符号链接（对应 lstat）。
lfs.touch(path [, atime [, mtime]])
修改文件的访问 / 修改时间（Unix 秒）。省略时用当前时间。
lfs.link(old, new [, symlink])
创建链接。symlink=true 创建符号链接，否则硬链接。
lfs.lock(filehandle, mode [, start, length])
对 io.open 拿到的文件句柄加锁。mode 为 "r"（共享）或 "w"（独占）。start / length 可选， 指定加锁的字节范围（默认整文件）。返回 true 或 nil, err。
lfs.unlock(filehandle [, start, length])
释放 lock 拿到的文件锁。start / length 与 lock 时保持一致。返回 true 或 nil, err。
lfs.lock_dir(path [, seconds_stale])
跨进程目录锁（在目录下创建 lockfile.lfs 软链接）。 seconds_stale 默认 INT_MAX，超过这个秒数视为陈旧锁可直接抢占。 返回带 :free() 方法的对象——调 :free() 才真正释放锁。
lfs.setmode(filehandle, mode) Windows only
设置文件句柄为 "binary" 或 "text" 模式。仅 Windows 有意义； Android / Linux 上是 no-op，永远返回 true, "binary"。
lfs._VERSION
版本字符串常量，当前为 "LuaFileSystem 1.9.0"。

目录迭代器对象

lfs.dir(path) 在 for 循环里用最方便，但它实际返回三个值： iter_func, dir_obj, nil。dir_obj 上还有手动控制的方法， 适合需要中途中断、复用的场景：

dir_obj:next()
取下一个条目（字符串）。到末尾返回 nil。
dir_obj:close()
主动释放底层 DIR* 句柄。for 循环正常结束或遇到 GC 时会自动 close， 通常无需手动调。

常用片段

-- 递归创建目录 local function mkdirs(path) local cur = "" for part in path:gmatch("[^/]+") do cur = (cur == "") and ("/" .. part) or (cur .. "/" .. part) local a = lfs.attributes(cur) if not a then lfs.mkdir(cur) end end end -- 递归列文件（深度优先） local function walk(dir, fn) for name in lfs.dir(dir) do if name ~= "." and name ~= ".." then local p = dir .. "/" .. name local a = lfs.attributes(p) if a and a.mode == "directory" then walk(p, fn) else fn(p, a) end end end end -- 文件存在 + 是目录 local function isdir(p) local a = lfs.attributes(p) return a and a.mode == "directory" end -- 取文件大小（不存在返回 -1） local sz = lfs.attributes("/sdcard/foo.png", "size") or -1 -- 手动控制目录迭代器（提前中断） local iter, dir_obj = lfs.dir("/sdcard/Pictures") while true do local name = iter(dir_obj) if not name then break end if name:match("%.png$") then dir_obj:close() -- 找到第一个 png 立即收工 return "/sdcard/Pictures/" .. name end end -- 文件区间独占锁（比如配置文件读写） local f = io.open("/data/local/tmp/easylua/cfg.lua", "r+") if lfs.lock(f, "w") then -- ... 安全读写 lfs.unlock(f) end f:close() -- 跨进程目录锁，30 秒陈旧时强抢 local lk, err = lfs.lock_dir("/data/local/tmp/easylua", 30) if lk then -- ... 临界区 lk:free() else Log("已有别的进程在跑: " .. err) end
实现说明：LuaJIT 的 io 句柄是 IOFileUD（首字段是 FILE*）， 与 Lua 5.2+ 的 luaL_Stream 二进制不一致。我们在编译 lfs.c 时用 -DLFS_USE_LUAJIT_FILE=1 强制走 5.1 风格 FILE** 兼容层。 这个细节用户无需关心，lfs.lock(io.open(...)) 直接能用。

cjson JSON 编解码（内置纯 Lua）

easyLua 内置一份纯 Lua 的 cjson 实现，API 与 lua-cjson 兼容。 启动时自动注册为全局 cjson 表，require("cjson") 也能拿到同一份。 典型用途：HTTP 接口序列化、配置存档、调试输出。

cjson.encode(v, pretty)
把任意 Lua 值序列化为 JSON 字符串。pretty = true 时输出多行缩进 （2 空格，按 key 字典序排序，方便 diff），默认紧凑单行。
支持的类型：nil → null， boolean / number / string 直转， table 自动判别 array（连续 1..n）或 object（其它）， function / userdata / cdata 等转字符串占位（用 tostring）。
cjson.decode(s)
把 JSON 字符串解析为 Lua 值。出错抛 error，建议外面套 pcall 兜底。 支持 \uXXXX 转义到 UTF-8（仅 BMP 平面），尾部空白 OK，多余内容会报错。
cjson.null
表示 JSON null 的占位单例。decode 时遇到 null 返回这个值； encode 时显式塞入它会输出 null。和 nil 区别：表里 x = nil 等于"没有 x 字段"，table 序列化时会丢失；想保留就用 x = cjson.null。
cjson._VERSION
版本字符串，当前为 "easylua-cjson 1.0 (pure-lua)"。

print 已增强：table 自动展 JSON

runtime.lua 启动时把全局 print 重新包装：参数中的 table 会被 cjson.encode(t, pretty=true) 展开多行输出，其它类型保持原样。 意味着你不需要再手写 for k,v in pairs(t) do ... end 调试表内容。

-- 旧行为：print({a=1,b=2}) 输出 "table: 0x..." -- 新行为：自动展开成多行 JSON print({ name = "alice", age = 28, tags = {"vip", "new"} }) -- 输出： -- { -- "age": 28, -- "name": "alice", -- "tags": [ -- "vip", -- "new" -- ] -- } -- 混合参数：table 展开，其它原样 print("resp =", cjson.decode('{"ok":true,"data":[1,2,3]}')) -- 输出：resp = { ... 多行 JSON ... }

常用片段

-- 1) 简单存档（替代 io.open + loadstring） local function save(path, t) local f = io.open(path, "w") f:write(cjson.encode(t, true)) -- pretty 方便手动看 f:close() end local function load(path) local f = io.open(path, "r") if not f then return {} end local data = f:read("*a") f:close() local ok, v = pcall(cjson.decode, data) return ok and v or {} end -- 2) HTTP 请求 + 解析 local body = Exec("curl -sS https://httpbin.org/json") local ok, doc = pcall(cjson.decode, body) if ok then print(doc.slideshow.title) -- 直接当 lua 表用 end -- 3) 上报事件（紧凑格式更省字节） local payload = cjson.encode({ device = GetModel(), battery = GetBattery(), ts = os.time(), }) Exec("curl -sS -X POST https://api.example.com/event -d '" .. payload .. "'") -- 4) 区分 null 与 nil local t = { keepNull = cjson.null, dropMe = nil } print(cjson.encode(t)) -- {"keepNull":null}
选择紧凑还是 pretty：写文件存档建议 pretty=true 方便手动 diff； 发 HTTP / 写日志则用紧凑（默认）省字节。print 那条路径强制 pretty—— 反正只在调试时看。
性能：纯 Lua 实现比 lua-cjson 的 C 版慢 5~10 倍。每次几 KB 的 payload 完全无感； 若你的脚本在热路径里序列化几 MB 的数据就需要换实现了——不过那种场景很罕见。
